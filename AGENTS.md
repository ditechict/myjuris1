# AI Agent Instructions for myjuris1

## Purpose

This repository is a Cloudflare Workers web app built with `@tanstack/react-start`, `@tanstack/react-router`, React, TypeScript, Tailwind, and Supabase auth.

## How to run

- `npm install`
- `npm run dev` to start the local Vite app
- `npm run build` to build the app
- `npm run lint` to lint the project
- `npm run format` to format code
- `npx wrangler@3 publish --config wrangler.jsonc` to deploy

## Key architecture

- `src/start.ts`: application start configuration, middleware pipeline, Supabase auth attachment, error handling
- `src/server.ts`: Cloudflare Workers `fetch` entrypoint and SSR error normalization
- `src/router.tsx`: router creation using `@tanstack/react-router`
- `src/routeTree.gen.ts`: generated route tree; do not edit manually
- `src/routes/`: route definitions and pages, including auth and authenticated sections
- `src/components/ui/`: reusable UI components and Radix wrappers
- `src/integrations/supabase/`: Supabase auth integration and client setup

## Important conventions

- TypeScript is strict; source files include `src/**/*.ts`, `src/**/*.tsx`, `vite.config.ts`, and `eslint.config.js`
- Module alias `@/*` maps to `./src/*`
- `routeTree.gen.ts` is auto-generated by TanStack Router and should be excluded from manual edits
- `wrangler.jsonc` is the Cloudflare Workers deployment config; `src/server.ts` is the Workers entrypoint
- Use existing UI abstraction components rather than introducing ad hoc Radix/Tailwind wrappers when possible

## Files to avoid editing directly

- `src/routeTree.gen.ts`
- generated artifacts not under `src/routes/`

## Notes for AI agents

- Prefer small, incremental changes and keep the existing middleware/error handling flow intact
- Restore repository patterns when adding features: route-based pages, Supabase auth integration, and TypeScript+Tailwind styling
- If adding new pages, follow the route file structure in `src/routes` and update types through the existing route generation process
